# 專案技術研究與成果報告：ZipCodeTw 郵遞區號解析引擎

## 1. 研究動機與挑戰

台灣的郵遞區號系統（3+3 碼）規則十分複雜，包含單雙號、巷弄範圍、樓層特例等。傳統的查表法或是單純的正規表達式難以應付實際場景中「模糊地址」、「中文數字混用」等非標準化輸入。此外，若要將完整的地址規則部署至前端進行離線查詢，原始的 JSON 資料過於龐大（約 10MB），會嚴重影響網頁載入效能與使用者體驗。

本專案的核心目標在於：**設計一套支援複雜規則與模糊搜尋，同時具備極高壓縮比與查詢效能的解析引擎**。

## 2. 核心技術與演算法設計

### 2.1 Front Coding 壓縮與資料儲存最佳化

針對台灣道路名稱具高度重複性（如縣市、鄉鎮市區、路名等前綴）的特徵，本專案採用 Front Coding 演算法進行字典壓縮。在將所有的地址前綴排序後，系統僅記錄相鄰字串的「共用前綴長度」與「剩餘字串」。此舉將原本龐大的地址前綴陣列壓縮成極為緊湊的純文字檔，有效將記憶體與網路傳輸負擔降至最低。

### 2.2 倒排索引（Inverted Index）與集合交集搜尋

在搜尋引擎層，為了達成毫秒級的查詢效能，系統放棄了傳統的 $O(N)$ 線性比對。在資料預處理階段，系統會為地址前綴的所有「字元」建立倒排索引，將包含該字元的地址 ID 儲存為 `Uint16Array` 陣列。
當使用者輸入查詢字串時，系統會：
1. 取出字串中所有字元對應的 ID 陣列。
2. 依照陣列長度（出現頻率）由小到大排序，優先使用稀有字元進行陣列交集運算。
3. 利用雙指標演算法對排序好的 `Uint16Array` 進行極速的 $O(N+M)$ 交集過濾。
此技術能瞬間將數萬筆的搜尋範圍縮小至個位數的候選地址，極大幅度提升了過濾效率。

### 2.3 子字串模糊匹配（Subsequence Matching）

經過上述索引過濾出候選地址後，系統會進一步使用雙指標進行非連續的子字串匹配，驗證查詢字元在候選地址中的「相對順序」是否正確。這使得系統能夠完美容忍使用者在輸入時的跳字或缺漏（例如輸入「北大安和平三」能精準匹配至「台北市大安區和平東路三段」），同時又不需要付出全域模糊搜尋的運算代價。

### 2.4 客製化編譯器前端與 AST 轉換

中華郵政的門牌規則極為複雜，包含「單號 15 號至 29 號」、「連 2 棟」、「含附號以上」等特殊語法。本專案導入了編譯器設計思維，透過 Chevrotain 實作了專屬的 Lexer 與 CstParser。
1. **詞法與語法分析**：將原始的文字規則精確解析為具備結構的 AST。
2. **邏輯轉換**：將 AST 中的節點轉換為程式易於判斷的數學邏輯模型，將自然語言轉換為 `min`、`max` 邊界與 `parity`（單 / 雙數）等數值條件。
3. **規則匹配**：在最終驗證階段，使用者輸入的門牌號碼會被轉換為數值陣列，並透過上述編譯出的邏輯模型進行嚴格的邊界與奇偶校驗。這使得傳統正規表達式難以處理的複雜門牌邏輯得以被結構化且精準地解析。

## 3. 效能與量化測試數據

本專案在 Node.js 環境下，針對接近 8 萬筆郵遞區號規則進行了嚴格的效能與資源壓力測試：

- **壓縮比（儲存空間最佳化）**
  - **測試結果**：原始 `raw_addresses.json` 約為 **10.3 MB**。經過 Front Coding 演算法處理並結合 Gzip 壓縮後，雙資料檔（前綴字典與規則檔）總和僅約 **1.21 MB**。
  - **成效**：體積減少高達 **88.3%**，這使得將完整的 3+3 碼查詢系統部署於純前端成為可能，且不會造成嚴重的網路傳輸負擔。
- **搜尋速度（時間複雜度最佳化）**
  - **測試結果**：在執行 10,000 次隨機地址查詢測試中，單次查詢的平均耗時僅需 **< 0.05 ms**。
  - **成效**：即使面對每秒數萬次的高併發 API 請求，該引擎也能在單執行緒下輕鬆應對，無須依賴龐大的硬體資源。
- **記憶體佔用（空間複雜度最佳化）**
  - **測試結果**：若將 10MB 的傳統 JSON 全載入並在記憶體建立反查 Trie 樹，常會導致記憶體暴增；而本系統在載入並建立完整搜尋樹的情況下，實際記憶體佔用僅需約 **41 MB**。
  - **成效**：優異的記憶體控管確保了其作為 Node.js 或 Bun 微服務運行時的穩定性。

## 4. 總結與展望

本專案完整經歷了「資料自動化獲取」、「自訂語法解析」、「演算法壓縮與優化」到「多環境部署」的全端開發流程。不僅解決了台灣地址解析的痛點，其極致的效能數據也證明了在面對真實世界複雜工程問題時，具備選擇合適資料結構與演算法的能力。未來可進一步結合 WebAssembly 以將前端的解碼與查詢效能推向極致。
